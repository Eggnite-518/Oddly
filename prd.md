# **产品需求文档 (PRD)：Oddly - 你的潜意识侦探**

**文档版本：** V1.1
**更新日期：** 2026-05-14
**产品代号：** Oddly
**项目性质：** 个人项目 · 供开发者本人使用 · 无商业化目标

---

## **一、 产品概述与功能定位**

### **1.1 产品概述**
Oddly 是一款帮助用户捕捉日常生活中转瞬即逝的"奇怪想法"，并通过 AI 引导反思与数据可视化，帮助用户探索潜意识、洞察核心价值观与真实性格的自我探索类 App。

### **1.2 功能定位**
- **非医疗化的心理探索工具**：有别于传统的 CBT 干预应用，Oddly 强调"探索乐趣"而非"症状治疗"。
- **智能化的碎片日记**：有别于 Flomo 等纯效率工具，Oddly 具备主动解析能力，充当用户的"心理学镜子"。
- **核心价值主张**："接纳你的奇思妙想，遇见水面之下的自己。"

---

## **二、 目标人群与痛点分析**

### **2.1 目标人群**
本版本为单用户个人项目，开发者即用户。产品原型人群画像：
- 内心戏丰富、经常产生各种联想的高敏感人群
- 需要捕捉灵感的创意工作者
- 热衷于自我成长与心理学探索的人群

### **2.2 用户痛点**
- **想法流失**：脑海中经常冒出有趣的、荒谬的想法，但转瞬即逝
- **自我认知模糊**：不知道这些想法意味着什么，缺乏系统性的自我洞察
- **隐私与羞耻感**：有些想法过于私密，不敢发在社交媒体或写进普通日记

## **三、 核心功能描述**

Oddly 的核心功能围绕 **Catch（捕捉） - Trace（溯源） - Mirror（镜像）** 闭环展开。

### **3.1 Catch 层：无摩擦极速捕捉**
- **闪念输入舱**：App 首页即全屏输入框，支持文字、语音（自动转文字）
- **情境自动抓取**：在用户授权下，记录想法产生时的时间、地理位置、当前天气
- **桌面小组件（Widget）**：实现不打开 App 即可一键记录（Phase 3 实现）

### **3.2 Trace 层：AI 深度溯源与解构**

Trace 层是产品核心体验，分为三个递进阶段：

**阶段一：苏格拉底式追问（对话）**
- 用户提交想法后，同步触发 AI 分析，在详情页展示第 1 个引导性问题
- 用户可选择回答或跳过；AI 根据回答内容自行判断是否继续追问
- 追问上限为 3 轮，AI 认为已足够深入时主动结束对话

**阶段二：洞察卡片（Insight Card）**
- 追问结束（或用户主动跳过全部追问）后，AI 自动生成一张「洞察卡片」
- 卡片内容结构：
  - **深层解读**：用 2-3 句话解释这个想法背后可能隐藏的心理动因（例如：对控制感的渴望、未被满足的依恋需求等）
  - **心理学视角**：引用 1 个专业心理学框架或概念作为参考视角（荣格阴影、依恋理论、防御机制、马斯洛需求层次、IFS 内部家庭系统等），用通俗语言解释，非诊断性表达
  - **行动指南**：给出 1-2 个具体可执行的小行动或反思练习（例如："今晚睡前，试着写下三件今天让你感到失控的小事" / "下次这个想法出现时，先做5次深呼吸，然后问自己：现在我真正需要的是什么？"）
- 卡片风格：温暖、探索性语气，明确标注「这只是一种探索视角，不是诊断」

**阶段三：情绪标签（随洞察卡片一并生成）**
- AI 在同一次 API 调用中，随洞察卡片一起返回情绪标签字段（如：焦虑、好奇、悲伤、渴望等）
- 不作为独立功能入口，仅作为数据字段存储，在详情页轻量展示
- 用户可手动修改标签
- 主要用途：Phase 2 思维星图的节点着色分组 + Timeline 情绪筛选

**完整单条想法交互流程：**
```
首屏输入框
  └→ 提交
       ├→ [后台] 抓取时间 / 天气 / 位置
       └→ [同步] 跳转到详情页
            └→ AI 第1轮追问出现
                 ├→ 用户回答 → AI 判断是否继续（最多3轮）
                 └→ 用户跳过
                      └→ [无论哪种] 对话结束后生成「洞察卡片」
                           └→ 停留在详情页，展示完整记录 + 对话 + 洞察卡片
```

### **3.3 Mirror 层：模式识别与可视化**
- **Constella（思维星图）**：将所有想法转化为 2D 节点图，AI 自动将底层逻辑关联的想法连线
- **潜意识洞察报告（周/月度）**：基于积累数据，生成高频词云与心理学镜像分析

## **四、 功能实现规划 (Roadmap)**

### **Phase 1：MVP（2-4 周）**
**目标**：跑通"记录 → AI 追问 → 洞察卡片"完整核心链路

- 全屏文字输入 + 语音输入（自动转文字）
- 接入 DeepSeek API，实现：想法输入 → AI 苏格拉底式追问（最多3轮，AI自决）→ 洞察卡片生成
- 洞察卡片包含：深层解读 + 1个心理学视角 + 1-2个行动建议
- 详情页展示完整记录、对话历史、洞察卡片
- 列表式时间轴回顾（Timeline）
- 本地 SQLite 数据存储
- 情绪标签随洞察卡片同一次 API 调用返回，详情页展示，可手动修改

### **Phase 2：核心价值放大（第 2-3 个月）**
**目标**：上线可视化分析

- 2D"思维星图"（Graph View），实现想法之间的关联连线
- 情境自动记录（天气、时间段）
- 生成第一版周度/月度洞察报告（调用 DeepSeek 生成）
- 情绪标签自动提取

### **Phase 3：体验打磨（第 4-6 个月）**
**目标**：提升日常使用的流畅感

- 云端备份（Google Drive API 备份本地 SQLite 文件，非自建服务端）
- Android 桌面小组件（Widget）
- Embedding 语义自动关联（调用 DeepSeek Embedding API，在本地计算相似度后存库）

## **五、 技术方案**

### **5.1 前端技术栈**
- **框架**：Flutter（Android 优先，minSdk 24+）
- **本地数据库**：SQLite + Drift ORM
- **2D 可视化**：flutter_graph_view 或 CustomPainter 自定义实现
- **语音转文字**：speech_to_text 插件

### **5.2 后端**
**无自建后端**。个人单设备使用，所有数据本地存储，通过 HTTP 直连 LLM API。

### **5.3 AI 方案**
- **大语言模型**：DeepSeek API（DeepSeek-V3 用于日常追问，DeepSeek-R1 用于洞察报告生成）
- **接口形式**：客户端直接调用 DeepSeek OpenAI 兼容接口
- **API Key 管理**：存储于本地 `.env` 文件，不提交 Git
- **Prompt 设计**：System Prompt 设定 AI 扮演"客观、温暖、富有洞察力的心理学向导"，避免说教或医疗诊断
- **Embedding（Phase 3）**：调用 DeepSeek Embedding API，计算想法间语义相似度，超过阈值自动在本地数据库建立关联

### **5.4 数据存储设计**
使用 SQLite，无需图数据库，通过关系表实现想法关联：

```
thoughts         (id, content, created_at, location, weather, audio_path)
thought_tags     (thought_id, tag_name, is_auto, emotion_score)
ai_conversations (id, thought_id, question, answer, created_at)
thought_links    (thought_id_a, thought_id_b, similarity_score, created_at)
```

### **5.5 隐私方案**
- **本地优先**：所有数据默认存储在设备本地，无服务端
- **无 E2EE**：个人项目无需端对端加密
- **云备份（可选）**：Phase 3 支持将 SQLite 文件备份至 Google Drive，用户主动触发

## **六、 UI 设计规范**

### **6.1 设计风格**
**治愈温暖手绘风 · 方案 A「暖羊皮纸」**

### **6.2 配色方案**
| 用途 | 颜色 | Hex |
|------|------|-----|
| 页面背景 | 纸白 | `#FAF6EE` |
| 卡片背景 | 奶油 | `#FFF8EC` |
| 主强调色 | 焦糖 | `#C4874A` |
| 深强调色 | 深棕 | `#8B5E3C` |
| 浅辅助色 | 浅沙 | `#D4B896` |
| 主文字色 | 墨棕 | `#3D2B1F` |

### **6.3 字体**
- **正文**：Nunito（英文圆体）/ HarmonyOS Sans Round（中文）
- **标题 / 手写感标签**：Caveat（英文）/ 马善政体 Ma Shan Zheng（中文）

### **6.4 设计语言规则**
- **仅浅色模式**，不开发深色模式
- **卡片**：轻微不规则圆角 + 细描边，模拟手绘卡片质感
- **图标**：粗细不均的线条图标，保留手绘笔触感
- **装饰元素**：散落的小圆点、短曲线等涂鸦线条作为背景纹理点缀
- **动效**：弹簧物理动效（spring animation），卡片入场带轻微摆动
- **无全页插画**：手绘感仅体现在图标、装饰线条、字体，不绘制全页插画
