# 产品需求文档 (PRD)：Oddly - 你的潜意识侦探

**文档版本：** V2.0
**更新日期：** 2026-06-03
**产品代号：** Oddly
**项目性质：** 个人项目 · 作品集展示 · 无商业化目标

---

## 一、产品概述与定位

### 1.1 产品概述

Oddly 是一款帮助用户捕捉日常生活中转瞬即逝的奇思妙想，并通过 AI 引导反思与模式识别，探索潜意识、认识自我的工具。

它不是日记，也不是 CBT 干预工具——它是一面镜子，温和地映出水面之下的你。

### 1.2 核心价值主张

> **接纳你的奇思妙想，遇见水面之下的自己。**

- **非诊断性**：AI 始终以探索性而非结论性语气表达，明确标注"这只是一种可能的视角"
- **本地优先**：所有数据默认仅存本机，用户完全掌控隐私
- **轻量介入**：捕捉成本极低，AI 分析自动触发，无需用户主动填表

### 1.3 目标用户

内心戏丰富、经常产生各种联想的高敏感人群；热衷自我成长与心理学探索的群体；需要一个私密、无评判空间来整理内心的人。

---

## 二、核心功能闭环

Oddly 围绕 **Catch → Trace → Mirror → Act** 四层闭环展开：

```
捕捉想法（Catch）
    ↓
AI 追问 + 洞察卡片（Trace）
    ↓
暗流识别 + 认知模式统计（Mirror）
    ↓
行动建议收藏与落地（Act）
    ↓（循环）
```

## 三、功能详述

### 3.1 Catch 层：无摩擦极速捕捉

**首页输入区**
- App 首页即输入框，打开不自动弹出键盘，用户主动点击才聚焦
- 光标起点与「此刻在想什么？」提示语对齐（Stack 布局，不偏移）
- 键盘弹出时，统计卡片和行动卡片自动收起，键盘收回后平滑还原
- 底部字数统计实时更新；提交按钮点击后有轻触觉反馈（HapticFeedback）

**输入方式**
- 文字输入（多行，支持换行）
- 语音输入：长按麦克风录音，松手自动调用阿里云 NLS 转写；转写完成追加到文本末尾

**情境自动抓取**（与页面跳转并行，不阻塞用户操作）
- 时间段（上午 / 下午 / 傍晚 / 深夜）
- 天气（调用 Open-Meteo API）
- 地理位置城市名（IP 定位）

**首页统计卡片**（有数据后显示）
- 本周记录数 / 总计数 / 连续记录天数 / 最近情绪 / 上次记录时间

**首页行动提醒卡片**（有待完成行动时显示）
- 显示当前置顶的一条待完成行动建议
- 点击内容区：跳转至来源洞察卡片
- 右侧勾选按钮：标记完成
- 底部「跳过」：临时切换下一条（不更改数据库状态）
- 底部「所有行动」：进入行动清单页

---

### 3.2 Trace 层：AI 深度溯源与解构

#### 3.2.1 苏格拉底式追问

用户提交想法后立即跳转详情页，AI 自动发起第一轮追问。

**追问规则**
- 每次只问 1 个问题，必须呼应用户原话中的具体词语，≤25 字
- AI 根据回答判断是否继续，最多追问 3 轮
- 用户可随时选择跳过某轮或跳过全部直接生成洞察

**Prompt 设计原则**：温暖、好奇、不评判、不暗示、不给答案——AI 只是好奇地问。

#### 3.2.2 洞察卡片（Insight Card）

追问结束后自动生成，包含七个结构化字段：

| 字段 | 说明 |
|------|------|
| **深层解读** | 2-3 句，探索心理动因；必须呼应用户原话中的具体细节；末尾表明这是探索视角 |
| **心理学框架** | 选 1 个最贴切的框架，先解释框架本身，再说明与此想法的关联 |
| **行动指南** | 1-2 个今天就能做的具体行动，包含时间锚点 + 具体动作 + 具体对象 |
| **情绪标签** | 2-4 个词，从预设 17 个情绪词中选，支持手动修改 |
| **核心模式** | 4-12 字名词短语，提炼最深层的心理动因 |
| **推荐视角** | 另外 2 个推荐框架，供「换个角度看」使用 |
| **思维惯性** | 0-2 个认知偏误标签，含 AI 个性化解释（见 3.2.4）|

**可选心理学框架（共 11 个）**：
荣格阴影、依恋理论、防御机制、马斯洛需求层次、IFS 内部家庭系统、客体关系理论、存在主义焦虑、情绪调节理论、叙事治疗、自我同情、**阿德勒个体心理学**

> **阿德勒专属视角**：目的论切入（不问"为什么产生这个情绪"，而问"这个模式在帮你达到什么目的"），行动建议体现课题分离或贡献感。

#### 3.2.3 多视角切换

- 洞察卡片底部导航栏显示「当前框架 · 1/N」
- 点击右箭头：调用 AI 用下一个推荐框架重新生成洞察（懒加载，按需生成，已生成的缓存）
- 点击左箭头：切回已生成的视角
- 每个视角包含独立的深层解读、心理学框架、行动指南

#### 3.2.4 思维惯性标签

**8 类认知偏误**：灾难化联想、非此即彼、读心术、把一切揽到自己身上、用感受下判断、以偏概全、过滤负面信息、应该句式

- 每个 tag 可点击，弹出底部卡片展示：
  - 该惯性的通用心理学解释
  - AI 针对这条想法的**个性化说明**（为什么在这里触发了这个惯性）
- 同一个思维惯性在多条想法中累计出现 ≥3 次时，展示高频提示条，点击跳转 Mirror 查看关联想法

---

### 3.3 Mirror 层：模式识别

Mirror 页展示跨多条想法提炼出的长期模式，分两个维度：

#### 3.3.1 暗流（Currents）

「暗流」是跨越多条记录、反复涌现的深层心理模式。

- 每次生成洞察卡片后，后台静默提取/合并暗流（不阻塞主流程）
- 每条暗流包含：名称、描述、强度（出现次数）、关联心理学理论、关联情绪词
- 点击暗流：展示所有触发该暗流的关联想法卡片列表

#### 3.3.2 认知模式统计（Patterns）

汇总所有想法中出现过的思维惯性，按频次排列。

- 每条展示：惯性名称、出现次数（去重 thought_id 数）、首次出现时间
- 高频标记：出现 ≥5 次高亮显示
- 点击：展示所有触发该惯性的关联想法卡片列表
- **数据一致性保证**：删除想法时同步移除关联，启动时校验并修复孤立 ID

#### 3.3.3 Mirror 页入口

- 顶部「想试试的事」胶囊按钮，显示待完成行动数，点击进入行动清单页

---

### 3.4 Act 层：行动闭环

这是连接「洞察」与「现实改变」的一步。

#### 3.4.1 收藏行动建议

- 洞察卡片的每条行动建议右侧有书签图标
- 点击书签：将该行动存入行动清单，图标变为实心
- 已完成的行动：图标变为勾选状态
- 未收藏：空心书签

#### 3.4.2 首页行动提醒卡片

- 有待完成行动时，首页自动显示提醒卡片（置顶的一条）
- 完成后自动切换到下一条待完成行动
- 「跳过」不改变数据库状态，只临时切换展示

#### 3.4.3 行动清单页（想试试的事）

入口：首页行动卡片「所有行动」按钮 / Mirror 页顶部胶囊按钮

- **待完成**区：列出所有 pending 状态行动，点击来源跳转对应洞察卡片
- **已完成**区：折叠展示已打勾的行动
- **长按删除**：长按任意行动项 → 确认弹窗 → 删除后洞察卡片书签状态同步恢复

---

### 3.5 Timeline：时间轴回顾

- 按时间倒序列出所有想法卡片
- **情绪标签筛选**：横向滚动展示常用标签；右侧固定漏斗图标点击展开全量标签（DraggableScrollableSheet + Wrap 布局）
- 记录新想法后自动刷新
- 点击卡片：跳转详情页（已有洞察直接展示，无需重新生成）

## 四、技术方案

### 4.1 技术栈

| 领域 | 方案 |
|------|------|
| 框架 | Flutter 3，Android 优先（minSdk 24+） |
| 状态管理 | Riverpod（StateNotifierProvider） |
| 本地存储 | SQLite（sqflite），无 ORM |
| AI | DeepSeek API（通过阿里云函数计算代理转发） |
| 语音转文字 | 阿里云 NLS（实时语音识别） |
| 天气 | Open-Meteo API |
| 定位 | IP 定位（无需 GPS 权限） |
| UI 风格 | 暖羊皮纸手绘风，CustomPainter 自定义图标 |

### 4.2 后端架构

无自建后端服务。AI 请求通过**阿里云函数计算**上部署的轻量代理转发至 DeepSeek API，代理层仅做鉴权和请求转发，不存储任何用户数据。

### 4.3 AI Prompt 架构

所有 Prompt 集中在 `oddly/lib/core/constants/prompts.dart`，分层设计：

| Prompt | 用途 |
|--------|------|
| `socraticQuestionSystem` | 生成第一轮追问 |
| `continueQuestionSystem` | 判断是否继续追问 + 生成后续问题 |
| `insightCardSystem` | 生成完整洞察卡片（7 个字段） |
| `alternativePerspectiveSystem` | 换角度重新生成洞察（3 个字段） |
| `personaExtractionSystem` | 提取/合并暗流 |

**Prompt 设计原则**：温暖、好奇、不评判、非诊断；禁用空洞表达（"关注内心""与自己和解"等）；行动必须包含时间锚点 + 具体动作。

### 4.4 数据库 Schema（v5）

```sql
-- 想法主体
thoughts (
  id, content, created_at,
  latitude, longitude, location_name,
  weather, temperature,
  emotion_tags, is_analyzed
)

-- AI 对话历史
ai_conversations (
  id, thought_id, round,
  question, answer, was_skipped, created_at
)

-- 洞察卡片（支持多视角，perspective_index 区分）
insight_cards (
  id, thought_id, interpretation,
  psychology_theory, psychology_explanation,
  action_guide, core_pattern, cognitive_patterns,
  recommended_frameworks, perspective_index, created_at
)

-- 想法关联（预留，用于未来语义关联）
thought_links (thought_id_a, thought_id_b, similarity_score, created_at)

-- 暗流
persona_currents (
  id, name, description, strength,
  linked_theories, linked_emotions,
  thought_ids, first_seen_at, last_seen_at
)

-- 思维惯性统计（thought_ids 去重，count = thought_ids.length）
cognitive_pattern_stats (
  name, count, thought_ids, first_seen_at, last_seen_at
)

-- 行动清单
action_items (
  id, thought_id, content,
  status (pending/completed),
  is_pinned, created_at, updated_at
)
```

### 4.5 隐私方案

- 所有用户数据（想法、对话、洞察）仅存储在设备本地 SQLite
- AI 分析时将想法内容发送至 DeepSeek（经代理），语音转写发送至阿里云 NLS
- 无用户账号体系，无服务端数据库，无任何追踪
- API Key 通过 `.env` 文件注入，不提交 Git，不打包进公开 APK

## 五、UI 设计规范

### 5.1 设计风格

**治愈温暖手绘风 · 暖羊皮纸**

### 5.2 配色

| 用途 | 颜色 | Hex |
|------|------|-----|
| 页面背景 | 纸白 | `#FAF6EE` |
| 卡片背景 | 奶油 | `#FFF8EC` |
| 主强调色 | 焦糖 | `#C4874A` |
| 深强调色 | 深棕 | `#8B5E3C` |
| 浅辅助色 | 浅沙 | `#D4B896` |
| 主文字色 | 墨棕 | `#3D2B1F` |

### 5.3 字体

- 正文：Nunito（Google Fonts）
- 标题 / 手写感：Caveat（英文）/ 马善政体（中文）

### 5.4 设计原则

- 仅浅色模式
- 卡片：轻微不规则圆角 + 细描边，模拟手绘卡片质感
- 装饰元素：散落小圆点（ScatteredDots）作为背景纹理
- 动效：AnimatedSize、AnimatedOpacity、SlideTransition，保持轻盈感
- 三屏结构：底部导航栏（Capture / Timeline / Mirror），IndexedStack 维持状态

## 七、已知约束与设计决策记录

| 决策 | 原因 |
|------|------|
| 不使用 Drift ORM，直接用 sqflite | 减少依赖，Schema 迁移更可控 |
| AI 请求经代理而非直连 | 规避客户端直接暴露 API Key 的风险 |
| Mirror 数据删除想法时同步修复 | 防止统计数字与实际卡片数不一致 |
| 首页不自动弹出键盘 | 降低进入 App 的压迫感，用户主动触发输入 |
| 行动「跳过」不写库 | 跳过是临时操作，不应影响数据状态 |
| 思维惯性 count 用 thoughtIds.length | 同一想法被多次分析时去重，避免数字虚高 |
